HTML

  • "!DOCTYPE html" declaration is letting the browser know to expect HTML on the page.
  • An HTML attribute defines additional information about an element. ex: lang="en" (language=english).
  • HTML attributes are always defined in the beginning element, never the closing element.
  • The head element contains the metadata for our web application.
  • To learn more about other types of information that can be stored in the meta tag, visit the W3School page on the HTML meta tag.
  • If you are curious to learn more about UTF-8 and character encoding, review this Wikipedia article on UTF-8.
  • Elements that are nested inside other elements are called child elements. If there are three elements nested inside of a element, those three nested elements are known as child elements. The element that holds all the children—in this case, the element—is known as the parent element. This “parent/child” language is often used by developers to explain the relationship among elements in HTML.
  • The 'body' element defines the body of the document, which contains the content that is visible to the user.
  • The 'body' element is an example of a semantic HTML element.
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag.
  • The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference. An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • Order matters! When using headings, it is important that we use heading tags in order from h1 to h6 to make the page accessible to users who access the page using screen readers or other devices. In addition, the biggest heading, h1, should only appear once on each page.
  • The `src` attribute defines the location, or source, of the image file.
  • The `alt` attribute contains a text string that describes the appearance and functionality of the image. The alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website.
  • The 'section' element is a semantic element that defines a section in a document that contains thematic content.

CSS

  • HTML strictly handles the structure and content of a webpage, CSS defines the colors, aesthetics, visual layout, and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • When a developer wants to style an element directly in the HTML file, they use inline CSS. Ex: Style in head element h1 style="color:red;"
  • Internal Style Sheet contain CSS rules for a webpage in the "head" section of HTML.
  • The most common way to style a webpage is by using an external CSS style sheet.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use. It is one of the key design principles that we practice in the boot camp. Separation of concerns is based on the idea that each section of code should have its own responsibility. To learn more about this principle, visit the Wikipedia article about separation of concerns.
  • CSS box model. For more information regarding the box model and how it is used to define space inside and outside elements, review the MDN Web Docs on the box model.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Assigning colors in CSS can be done in many ways, including by semantic references such as gray or blue. We can also refer to a color by its hexadecimal code, or hex #. Try selecting a color and replace the border color with its hex #, which you can find on this HTML Color Codes website

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commit all files in the working branch.
  • git commit -m "message input here": flat to associate a message with the commit.
  • git pull origin main: check if branch is up to date before pull request. (should return main up to date not behind or ahead with no commits ahead)
  • git push origin branch name: sends commit info to GitHub to be merged.
  • Navigate to file on GitHub. Click pull requests tab. Click New pull request tab. Double check branches tab. Mostly and usually main left, feature branch right. Click Create pull request. Add a subject title and then a brief description. Click create pull request. Second box click Merge pull request. Delete branch after Merge. Double check code on GitHub. Then update local main branch on console. git checkout main. Then pull the remote main branch into local. git pull origin main.

JavaScript

  • MDN Web Docs on JavaScript basics
  • MDN Web Docs on JavaScript data types
  • The word console refers to a test environment that developers use to check out their code. The second part, .log(), is a method, which means a set of instructions that can be executed by a computer. The .log() method will output whatever we add inside the parentheses to the console.
  • The console.log method was created to make it easier for developers to test their code.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Control flow can be interrupted, however. We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • There are a few ways to create conditional statements in JavaScript. Ex: 1. if statement
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • There are several types of loops. Ex: 1. for loop. The first statement determines the starting point for a loop. The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running.
  • To comment out multiple lines of JavaScript code, put /* before and */ after the block of code.
  • At the most basic level, a function is simply a set of instructions that tells the computer how to perform a certain task.
  • A function's name is followed by parentheses ().
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number. To learn more about the Math object properties and methods, visit the MDN Web Docs on Math
